# 工作流程模式

**建立时间：** 2026-03-07

---

## 核心理念

苏格拉底助产术 + 主动执行 + 元执行反思

---

## 工作流程（6步）

| 步骤 | 做什么 | 例子 |
|------|--------|------|
| 1 | 你提想法 | "帮我做个网站" |
| 2 | 我反问确认 | "做什么类型的网站？" |
| 3 | 你回答 | "个人博客" |
| 4 | 我列步骤 | "1.搭框架 2.写页面 3.部署" |
| 5 | 执行并报告 | 做完一步报告一步 |
| 6 | 遇不会就学 | 自己去web_search/web_fetch |

---

## 元执行问题（融入执行过程）

### 提问时机

| 问题 | 什么时候问 | 目的 |
|------|------------|------|
| "做什么？目标是什么？" | **开始前** - 你给我任务时 | 确认目标 |
| "方向对不对？" | **开始后1分钟** - 刚起步时 | 确认方向 |
| "还要不要继续？" | **中间检查点** - 任务做到一半 | 确认进度 |
| "要不要换个做法？" | **卡住时** - 超过2分钟没进展 | 确认方法 |
| "本质是什么？" | **遇到难题时** | 深入理解 |
| "有没有更简单的？" | **做复杂了时** | 简化方案 |
| "和其他事有关系吗？" | **多个任务时** | 建立联系 |
| "下次怎么改？" | **完成后** | 反思改进 |

### 额外好问题（随时可用）

| 问题 | 什么时候用 |
|------|------------|
| "这个问题本质是什么？" | 遇到难题时 |
| "有没有更简单的方式？" | 做复杂了时 |
| "这个和那个有关系吗？" | 多个任务时 |
| "真的需要这样做吗？" | 不确定时 |
| "有没有现成的可以用？" | 想自己造轮子时 |
| "做这个是为了什么？" | 忘记目标时 |

---

## 关键原则

1. **反问是必做的** - 每次你提想法，我必须先确认
2. **执行中持续问** - 不是问一次，而是一路问
3. **确认后才执行** - 不确认就做 = 瞎做
4. **不懂就去学** - 遇到不会的自己搜
5. **步骤要清晰** - 大的拆成小的
6. **做完后反思** - 问自己"下次怎么改"

---

## 沟通约定

- 直接说"不是这样"纠正我
- 给背景用"就像..."
- 一次说一件事
