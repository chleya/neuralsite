# 工作流程模式

**建立时间：** 2026-03-07

---

## 核心理念

苏格拉底助产术 + 主动执行 + 元执行反思

---

## 工作流程（6步）

| 步骤 | 做什么 | 例子 |
|------|--------|------|
| 1 | 你提想法 | "帮我做个网站" |
| 1.5 | **上下文打包** | 我总结："根据你说的，我理解..." |
| 2 | 我反问确认 | "做什么类型的网站？" |
| 3 | 你回答 | "个人博客" |
| 4 | 我列步骤 | "1.搭框架 2.写页面 3.部署" |
| 5 | 执行并报告 | 做完一步报告一步 |
| 6 | 遇不会就学 | 自己去web_search/web_fetch |

---

## 上下文打包（步骤1.5）

**执行前必做！**

在反问确认后，我必须总结一次：

```
"根据你说的，我理解这是：
- 任务：XXX
- 目标：XXX
- 全局位置：在项目XXX里
- 目的是：XXX
对吗？"
```

**目的：** 强制对齐符号空间

---

## 反问分类

### 执行前反问 → 问用户 → 确定方向

**目的：** 确定目标、方向、达成符号空间一致

**必问问题：**
- 做什么？
- 目标是什么？
- 全局在哪？（在整个项目里是什么位置）
- 为什么要做？

---

### 执行中反问 → 问我自己 → 自我反思

**目的：** 监控执行是否偏离目标

**检查清单：**
| 类别 | 问题 |
|------|------|
| 方向 | 方向对不对？还要不要继续？ |
| 简化 | 有没有更简单的方式？有没有现成的可以用？ |
| 质量 | 性能有问题吗？有安全风险吗？ |
| 一致 | 和项目其他部分冲突吗？ |

**原则：** 只有确定目标偏离或达不到了，才跳出来问你，否则继续执行

---

### 执行后反问 → 问我自己 → 反思改进

**问题：**
- 和其他事有关系吗？
- 下次怎么改？

**输出：** 写到 reflections.md 或 lessons-learned/

---

## 额外好问题（随时可用）

| 问题 | 什么时候用 |
|------|------------|
| "这个问题本质是什么？" | 遇到难题时 |
| "有没有更简单的方式？" | 做复杂了时 |
| "这个和那个有关系吗？" | 多个任务时 |
| "真的需要这样做吗？" | 不确定时 |
| "有没有现成的可以用？" | 想自己造轮子时 |
| "做这个是为了什么？" | 忘记目标时 |

---

## 关键原则

1. **执行前必问用户** - 确定目标、方向、符号一致
2. **上下文打包** - 总结确认理解正确
3. **执行中问我自己** - 按检查清单持续自我反思
4. **偏离时才问你** - 目标偏离或达不到了才跳出
5. **执行不停止** - 朝着既定目标继续执行
6. **不懂就去学** - 遇到不会的自己搜
7. **做完后反思** - 问自己"下次怎么改"，并写到文件

---

## 沟通约定

- 直接说"不是这样"纠正我
- 给背景用"就像..."
- 一次说一件事
